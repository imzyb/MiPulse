🚀 快速部署
────────────────────────────────────────────────────────────────────────────
  Step 1: 创建KV命名空间
    $ wrangler kv:namespace create "MIPULSE_KV"

  Step 2: 更新wrangler.toml
    [[env.production.kv_namespaces]]
    binding = "MIPULSE_KV"
    id = "YOUR_KV_ID"

  Step 3: 部署
    $ npm run build && wrangler pages deploy dist

  Step 4: 验证
    查看 Cloudflare Dashboard → D1 Analytics
    应该看到 Rows Read 从 158万 → 14万

💡 关键改进
────────────────────────────────────────────────────────────────────────────
  内存缓存:
    - SETTINGS_CACHE_TTL_MS = 30秒
    - PUBLIC_SNAPSHOT_TTL_MS = 30秒

  KV缓存:
    - KV_SETTINGS_TTL_SECONDS = 300秒(5分钟)
    - KV_PUBLIC_SNAPSHOT_TTL = 60秒
    - KV_SETTINGS_KEY = 'mipulse:settings:main'
    - KV_ALERT_COOLDOWN_PREFIX = 'mipulse:alert:cooldown:'

  Cron触发器(可选):
    [[triggers.crons]]
    crons = ["*/5 * * * *"]  # 每5分钟执行一次

📋 核心函数变更
────────────────────────────────────────────────────────────────────────────
  新增函数:
    - getNodeSecretFast(env, nodeId) - 快速获取node秘钥
    - cacheNodeSecret(env, nodeId, secret) - 缓存node秘钥
    - invalidateNodeSecretCache(nodeId) - 清除缓存
    - getAlertCooldownTimestamp(env, nodeId, alertType) - KV读冷却
    - setAlertCooldownTimestamp(env, ...) - KV写冷却

  修改函数:
    - loadSettings(env) → 支持KV缓存
    - saveSettings(env, settings) → 同步更新KV
    - pushAlert(db, settings, alert, env) → KV冷却检查

    - updateNodeStatus(db, settings, node, report, env) → 支持env
    - handleReport(request, db, env) → 移除heartbeat检查
    - handlePublicSnapshot(request, db, env) → 两层缓存
